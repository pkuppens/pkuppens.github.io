Content Understanding

Foundry's one tool for turning messy documents, images, audio, and video into clean structured data. A whole exam sub-domain lives here.

The one-service story

Azure AI Content Understanding is a Foundry Tool that uses generative AI to read any content type and emit a structure you define. It replaces the old patchwork — Document Intelligence (a.k.a. Form Recognizer) for docs, Video Indexer for video — with one multimodal service.

Remember the four inputs Content Understanding handles Documents · Images · Audio · Video. If a question mentions extracting fields or insights from any of those four, Content Understanding is almost always the answer.

How it works — inputs → analyzer → structured output

StageWhat happens
InputsYour raw files — documents, images, audio, or video.
AnalyzerThe reusable config that defines how content is processed and what fields to pull. Use a prebuilt analyzer (invoices, contracts, call-center, media…) or build a custom one.
Field extractionProduces structured key-value pairs from your schema — no prompt engineering needed.
Structured outputClean JSON (for automation/analytics) or Markdown (for search / RAG ingestion).

Three ways it fills a field

A frequently-tested distinction — how the analyzer produces each field value:

Extract

Pull a value exactly as it appears (invoice date, total). Documents only.

Classify

Pick from predefined categories (call sentiment, document type) and route accordingly.

Generate

Create a new value freely (summarize a call, describe a video scene).

Two quality features worth naming

Confidence scoresA 0–1 reliability estimate per field. High scores enable straight-through processing (no human review).
GroundingPoints to where in the source a value came from, so a human can verify it fast.
Scenario → answer drills "Read thousands of invoices into fields" → Content Understanding (Extract).   "Summarize call recordings" → Content Understanding (Generate, audio).   "Describe scenes in a video" → Content Understanding (Generate, video).   "Route documents by type first" → Content Understanding (Classify).
Trap Old names still appear as distractors: Form Recognizer and Document Intelligence (documents only) and Video Indexer (video only). On AI-901, the multimodal, current answer is Content Understanding.
