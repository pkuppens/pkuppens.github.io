Agents in Microsoft Foundry

A single-agent solution is an explicit exam objective. Learn what an agent is, how tools and knowledge extend it, and recognize how one is created and called.

What an agent is

A plain model answers one prompt at a time and forgets. An agent wraps a model with a fixed identity and the ability to act. Four ingredients:

Model

The deployed foundation model that generates text and reasons.

Instructions

A persistent system prompt — role, rules, tone — applied on every turn without repeating it.

Tools

Actions beyond text: search the web, run code, query your APIs or data.

Knowledge

Grounding data the agent retrieves from — files, databases, web — so answers are factual.

The one-line definition Agent = model + instructions + tools + knowledge. If a scenario needs multi-step action, tool use, or consistent behaviour across a conversation, the answer is "an agent" — not a bare model call.

Model vs. agent — when to reach for which

ScenarioAnswer
One-shot "summarize this text"A model call
Assistant that keeps its persona across many turnsAn agent (instructions)
"Look up today's prices, then calculate a quote"An agent (tools)
"Answer only from our internal handbook"An agent (knowledge / grounding)

Creating a single agent

Two doors again. In the Foundry portal: Build → Agents → Create agent, give it a name and instructions, then chat with it in place. In the SDK, the shape you should recognize:

from azure.identity import DefaultAzureCredential
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import PromptAgentDefinition

project = AIProjectClient(
    endpoint=PROJECT_ENDPOINT,
    credential=DefaultAzureCredential(),
)

# Define the agent: a model + persistent instructions
agent = project.agents.create_version(
    agent_name="SupportBot",
    definition=PromptAgentDefinition(
        model="gpt-5-mini",
        instructions="You are a helpful support assistant. Be concise and cite policy.",
    ),
)

Talking to the agent (multi-turn memory)

An agent keeps history within a conversation. That's what makes turn two aware of turn one:

openai = project.get_openai_client(agent_name="SupportBot")
conversation = openai.conversations.create()      # holds the history

r1 = openai.responses.create(conversation=conversation.id,
                             input="Can I return an item after 30 days?")
r2 = openai.responses.create(conversation=conversation.id,
                             input="And if it was a gift?")   # remembers r1
Exam cues "Consistent persona without re-sending instructions" → agent instructions.   "Remembers earlier turns" → a conversation.   "Can take an action / call an API / search" → a tool.   "Answers grounded in our data" → knowledge.
Single-agent scope AI-901 tests a single-agent solution — one agent doing a job. Multi-agent orchestration exists in Foundry but is AI-102 territory. Don't overthink it on this exam.
