Microsoft Foundry: Portal → SDK

The core build loop of Domain 2. Deploy a model in the portal, then call it from a lightweight app with the Foundry SDK. Learn the flow and recognize the code — that's what the exam grades.

The mental model

Microsoft Foundry is the one place you go to build AI apps and agents on Azure. Four nested ideas, biggest to smallest:

LayerWhat it is
Foundry resourceThe Azure resource that hosts your AI work (an Azure AI services resource).
ProjectA workspace inside the resource. Your endpoint points at a project. This is your unit of work.
Model deploymentA specific model (e.g. a GPT chat model) made callable under a name you choose.
AgentA deployed model + instructions (+ optional tools & knowledge) that behaves consistently. (Lesson 03.)
Two doors, same house Everything in Foundry is reachable two ways: the Foundry portal (web UI, no code — great for deploying and testing) and the Foundry SDK (code — for building apps). The exam expects you to know which door fits the task.

Step 1 — Deploy a model in the portal

In the Foundry portal you pick a model from the catalog and deploy it. Once deployed, you land in the Build section with a playground where you can chat with the model — no code — to test prompts before writing an app. This is where you practice writing a good system prompt (role, rules, tone) and user prompt (the actual request).

Exam cue "Test a prompt quickly without writing code" → Foundry portal playground. "Build an application that calls the model" → Foundry SDK.

Step 2 — Call it from a lightweight app (Foundry SDK)

A "lightweight chat client" is exactly the phrase in the exam objectives. Here's the current shape (Python, azure-ai-projects v2.x). You do not need to write this from memory — you need to recognize each piece:

from azure.identity import DefaultAzureCredential
from azure.ai.projects import AIProjectClient

# The project endpoint you copied from the Foundry portal
PROJECT_ENDPOINT = "https://my-resource.ai.azure.com/api/projects/my-project"

# 1. Connect to the project (auth via `az login`, no API key in code)
project = AIProjectClient(
    endpoint=PROJECT_ENDPOINT,
    credential=DefaultAzureCredential(),
)

# 2. Get a client for the deployed model
openai = project.get_openai_client()

# 3. Send input, get a completion back
response = openai.responses.create(
    model="gpt-5-mini",            # your deployment name
    input="Summarize this policy in one sentence.",
)
print(response.output_text)

Three things the exam loves to probe in this snippet:

PieceWhy it matters
azure-ai-projectsThe Foundry SDK package. v2.x = the new Foundry API; v1.x = Foundry classic.
DefaultAzureCredentialKeyless auth using your signed-in identity (az login). Preferred over hard-coded API keys — ties to the privacy & security principle.
model="..."References your deployment name, not a raw model file. No deployment → nothing to call.
