Speech, Vision & Multimodal

The rest of Domain 2's implementation surface: hearing, speaking, seeing, and generating images — and the modern shortcut of doing several at once with a multimodal model.

Azure Speech (in Foundry Tools)

One service, three jobs. Keep the direction of the arrow straight:

CapabilityDirectionUse it for
Speech recognition (speech-to-text)audio → textTranscribe a meeting, take a voice command.
Speech synthesis (text-to-speech)text → audioRead an answer aloud, build a voice assistant reply.
Speech translationaudio → audio/text (other language)Live cross-language conversation.

Vision — two opposite tasks

Interpret an image

Read/understand an existing image: caption it, read its text (OCR), answer questions about it. An analysis task.

Generate an image

Create a brand-new picture from a text prompt. A generative task, done by an image-generation model.

Don't mix these up "Describe what's in this photo" = interpret (multimodal model). "Make a picture of a red bicycle" = generate (image-generation model). The verb in the question — describe/read vs create/make — is the tell.

Multimodal models — the modern shortcut

A multimodal model accepts more than one input type in a single prompt. AI-901 deliberately favours these over stitching separate services together:

ScenarioAI-901's preferred answer
Send a photo + a text question, reason over bothA deployed multimodal model (interpret visual input in a prompt)
Respond to a spoken promptA deployed multimodal model that handles speech
Generate a new image from textAn image-generation model
Just transcribe or synthesize speech, no reasoningAzure Speech
The mental split Need reasoning across modalities (image+text, speech+text)? → one multimodal model. Need a plain conversion (audio↔text)? → Azure Speech. Need a new picture? → an image-generation model.

And text analysis
