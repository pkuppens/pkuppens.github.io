AI-901 · Reference

Glossary

The canonical AI-901 vocabulary. Every lesson uses these names. Where Azure renamed something, the current name wins and the old name is flagged — that swap is a classic exam trap.

Naming churn Azure renames things constantly. Distractor answers on the exam are often old product names. Learn the current name; recognize the retired one. The most bitten items are marked below.

Platform & services

Microsoft Foundry
The unified platform for building, deploying, and managing AI apps and agents on Azure — portal, SDK, and a model catalog in one place. Formerly "Azure AI Foundry", earlier "Azure AI Studio". This rename is the whole reason AI-901 exists.
Foundry portal
The web UI where you deploy models, test prompts in a playground, and build/test agents without writing code.
Foundry SDK
The client library (Python) for calling deployed models and agents from a "lightweight client application". The exam tests recognizing its shape, not writing it from memory.
Azure AI services
The umbrella family of prebuilt AI APIs (vision, speech, language, etc.). Formerly "Cognitive Services" — a frequent distractor.
Azure AI Foundry Agent Service
The managed service for building agents — an LLM plus instructions, tools, and knowledge that can take actions to complete a goal. Domain 2 tests creating and testing a single-agent solution.
Content Understanding
Azure's multimodal information-extraction service in Foundry Tools — pulls structured data from documents, images, audio, and video. Absorbs the jobs of the old "Form Recognizer" → "Document Intelligence" and "Video Indexer".
Azure Speech
The service for speech-to-text (recognition), text-to-speech (synthesis), and speech translation, available in Foundry Tools.
Azure OpenAI / model catalog
Where foundation models (GPT-family chat, embeddings, image-gen, multimodal) are deployed and served through Foundry.

Model & generative concepts

Generative AI
Models that create new content (text, images, audio, code) from a prompt, rather than only classifying or scoring existing input.
Agentic AI
An AI system that plans and takes multi-step actions using tools and knowledge to reach a goal, not just answering a single prompt. The "agent" in Domain 2.
Multimodal model
A single model that accepts and/or produces more than one modality — e.g. reads an image and text in one prompt. AI-901 favours multimodal models over separate vision/speech services for many tasks.
Token
The unit a model reads and generates — roughly a word-piece. Prompt length and cost are measured in tokens.
Prompt (system vs. user)
The input to a model. The system prompt sets role, rules, and tone; the user prompt is the specific request. Writing both well is an explicit exam objective.
Completion
The model's generated output for a given prompt.
Grounding / RAG
Supplying a model with external, retrieved data at prompt time so answers cite real sources instead of hallucinating. (You build these already — Azure exposes it as "add your data" / agent knowledge.)
Temperature
A decoding parameter for randomness: low = deterministic/focused, high = creative/varied. A common "configuration parameters" question.
Top-p (nucleus sampling)
An alternative randomness control that samples from the smallest set of tokens whose probabilities sum to p.
Grounded / fine-tuning distinction
Grounding adds data at inference time; fine-tuning changes the model's weights by training on examples. The exam probes which to choose.

Workload vocabulary (Domain 1 matching)

Sentiment analysis
Text-analysis technique scoring text as positive / negative / neutral.
Entity detection (NER)
Identifying named things (people, places, dates, quantities) in text.
Key phrase / keyword extraction
Pulling the main talking points from text.
Summarization
Condensing text (extractive: pick sentences; abstractive: generate new).
Speech recognition vs. synthesis
Recognition = audio → text (speech-to-text). Synthesis = text → audio (text-to-speech).
Computer vision
Interpreting images/video: classification, object detection, OCR, captioning.
Image generation
Creating new images from a text prompt (a generative, not analytical, vision task).
Information extraction
Turning unstructured content (docs, images, audio, video) into structured fields — the Content Understanding domain.

Responsible AI (the six)

Fairness
Equitable treatment across groups; no systematic bias.
Reliability & safety
Consistent, robust behaviour that fails safely.
Privacy & security
Protecting personal data and honouring consent.
Inclusiveness
Working for people of all abilities and backgrounds.
Transparency
Making the system's behaviour and limits understandable.
Accountability
Human governance and responsibility for outcomes.
