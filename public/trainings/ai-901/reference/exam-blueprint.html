AI-901 · Reference

Exam Blueprint

The authoritative skills-measured outline, straight from Microsoft. Everything testable lives here.

Primary source   Official Study Guide — Exam AI-901 · Skills as of 2026-04-15 · Page updated 2026-06-24.

Pass mark

700 / 1000. A scaled score — not a raw percentage. Aim for ~85%+ on practice to clear it comfortably.

Format

~40–60 questions, ~45 min. Multiple-choice, multi-select, drag-and-drop, case-style. No live labs.

Assumed skills

Python syntax literacy, familiarity with Azure resources, and awareness of REST APIs / SDKs / CLIs. Mostly GA features.

Domain 1 · Identify AI concepts & capabilities 40–45%

Conceptual recognition. No portal clicks — just knowing what and which.

Describe principles of responsible AI

The six Microsoft principles — expect a scenario, pick the principle it violates or upholds:

PrincipleOne-line test
FairnessDoes it treat all groups equitably? (no biased outcomes)
Reliability & safetyDoes it behave consistently and fail safely under stress?
Privacy & securityIs personal data protected and consent respected?
InclusivenessDoes it work for people of all abilities/backgrounds?
TransparencyCan users understand how/why it made a decision?
AccountabilityAre humans responsible and is there governance/oversight?

Identify AI model components and configurations

  • Describe how generative AI models work (tokens, prompts, completions, context).
  • Identify an appropriate AI model based on capabilities (chat vs. embeddings vs. multimodal vs. image-gen).
  • Identify deployment options and configuration parameters (temperature, max tokens, top-p, deployment types).

Identify AI workloads

  • Match scenarios to workloads: generative & agentic, text analysis, speech, computer vision, information extraction.
  • Text analysis techniques: keyword extraction, entity detection, sentiment analysis, summarization.
  • Speech: recognition (speech-to-text) and synthesis (text-to-speech).
  • Vision: computer vision and image-generation models.
  • Information extraction from text, images, audio, and video.

Domain 2 · Implement AI solutions using Microsoft Foundry 55–60%

The heavy half. Recognize how you'd build things in Microsoft Foundry (portal + SDK + agents). You won't write code from scratch, but you must recognize correct steps, service names, and tools.

Generative AI apps and agents by using Foundry

  • Write effective system and user prompts.
  • Deploy a model and interact with it in the Foundry portal.
  • Build a lightweight chat client with the Foundry SDK.
  • Create and test a single-agent solution in the Foundry portal.
  • Build a lightweight client application for an agent.

Text and speech by using Foundry

  • Lightweight app that includes text analysis.
  • Respond to spoken prompts using a deployed multimodal model.
  • Build an app using Azure Speech in Foundry Tools.

Computer vision & image generation by using Foundry

  • Interpret visual input in prompts via a deployed multimodal model.
  • Create new visual outputs using generative (image) models.
  • Build a lightweight app with vision capabilities.

Information extraction by using Foundry

  • Extract from documents/forms using Azure Content Understanding in Foundry Tools.
  • Extract from images using Content Understanding.
  • Extract from audio and video using Content Understanding.
  • Build a lightweight app with information-extraction capabilities.
Where the points are Domain 2 outweighs Domain 1, and its centre of gravity is Microsoft Foundry — the portal, the Foundry SDK, single-agents, and Content Understanding for multimodal extraction. If you're short on time, over-invest here. Domain 1 (responsible AI + workload-matching) is cheap, memorizable points — bank them all.
