Access tiers vs. redundancy — two different axes

Tiers control how often you access data; redundancy controls how many copies exist and where. They're independent settings, and the exam loves mixing them up.

Identity, access & security

Azure splits what many clouds bundle into "IAM" into two layers: Microsoft Entra ID (identities live here) and RBAC (permissions get assigned here, at a scope).

Naming trap "Microsoft Entra ID" is the current name (formerly Azure Active Directory / Azure AD). Distractors on the exam still use the retired name — recognize it, don't pick it.

Security principles

Zero Trust — "never trust, always verify," no implicit trust from network location — is the philosophy. Defense-in-depth — layered controls across network, identity, app, and data — is how you implement it. Microsoft Defender for Cloud is the tool that scores and hardens your posture against both.
