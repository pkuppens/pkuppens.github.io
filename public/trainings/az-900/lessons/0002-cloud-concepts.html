Cloud Concepts

The cheapest points on the exam — pure vocabulary and definitions, no Azure specifics required. You already know these ideas from AWS/GCP; this lesson is about the exact phrasing Microsoft expects.

Shared responsibility model

One model, three service types, one rule that never changes: the customer always owns data, identities, and endpoints. Everything else shifts as you move right:

LayerOn-premIaaSPaaSSaaS
Data, devices, accessAlways customer
OS patchingCustomerCustomerMicrosoftMicrosoft
Runtime / middlewareCustomerCustomerMicrosoftMicrosoft
Application codeCustomerCustomerCustomerMicrosoft
Physical datacenterCustomerAlways Microsoft (in Azure)
Exam cue If a question asks "who is responsible for X, in every model" — the answer is either always the customer (data/identity/access/endpoints) or always Microsoft (physical security), never "it depends."

Pricing: CapEx vs. OpEx

CapEx — buy hardware upfront, depreciate it over years. OpEx / consumption-based — pay only for what you use, expensed as you go. The cloud's core economic pitch is shifting CapEx to OpEx.

Benefits, precisely defined

BenefitOne-line test
High availabilityDoes it keep running and meet its SLA despite a failure?
ScalabilityDoes capacity grow/shrink automatically with demand?
ReliabilityDoes it recover and keep functioning consistently over time?
PredictabilityCan you forecast cost and performance in advance?
Security & governanceAre policy, compliance, and access centrally controlled?
ManageabilityIs deploying/monitoring/maintaining resources easy at scale?

Serverless is a step beyond PaaS: zero capacity to provision at all, billed strictly per execution (e.g. Azure Functions), rather than a running app-service plan.
