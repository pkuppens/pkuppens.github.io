Management & Governance

Domain 3 covers four practical clusters: cost, governance/compliance, deployment tooling, and monitoring. Most questions are "which tool does X" — precision on tool boundaries is what scores points here.

Cost management

ToolWhen you use it
Pricing calculatorBefore deployment — estimate a hypothetical configuration
Azure Cost ManagementAfter deployment — analyze actual spend, set budgets/alerts
TagsMetadata for cost allocation, filtering, automation (same concept/name as AWS)
Exam cue "Before deploying" → Pricing calculator. "Resources are already running, show me the trend" → Cost Management. Confusing these two is a top-5 miss on practice exams.

Governance & compliance

ToolAWS/GCP analogPurpose
Azure PolicyAWS Config / Org PolicyEnforce/audit resource configuration rules
Resource lock(no direct equivalent)Block delete (CanNotDelete) or delete+edit (ReadOnly), independent of RBAC
Microsoft PurviewData catalog/lineage toolingDiscover, classify, and govern data across the estate
Classic pairing question RBAC controls who can act. Azure Policy controls what configuration is allowed, regardless of who deployed it. A resource lock blocks specific actions (delete/edit) regardless of role — even an Owner is blocked until the lock is removed.

Deployment & management tooling

ToolShape
Azure portalWeb GUI

Azure CLI / PowerShellSame tools, installed locally instead
Azure ArcExtends Policy/RBAC/tags to non-Azure servers and Kubernetes

BicepFriendlier syntax that compiles to the same ARM templates

Monitoring

ToolAWS/GCP analogPurpose
Azure Advisor(broad best-practice recommendations)Proactive tips across cost/security/reliability/performance
Azure Service HealthPersonal Health DashboardIs Azure itself having an issue affecting your resources?
Azure MonitorCloudWatch / Cloud MonitoringCollects your resources' own metrics/logs, drives alerts
↳ Log AnalyticsCloudWatch Logs InsightsKQL queries over collected logs
↳ Application InsightsX-Ray / Cloud TraceApp-level performance telemetry
↳ AlertsCloudWatch AlarmsThreshold-based automated notification
Exam cue "Is the platform broken?" → Service Health. "Is my resource broken?" → Monitor. "What should I change to do better?" → Advisor. Three different questions, three different tools.
