AZ-900 · Reference

Exam Blueprint

The authoritative skills-measured outline, straight from Microsoft. Everything testable lives here.

Primary source   Official Study Guide — Exam AZ-900 · Skills as of 2026-07-20 (the English exam updates on this date; the change log shows only minor wording changes vs. the prior version, no domain restructuring).

Pass mark

700 / 1000. A scaled score — not a raw percentage. Aim for ~85%+ on practice to clear it comfortably.

Format

45 minutes. Multiple-choice, multi-select, drag-and-drop, case-style. No live labs.

Assumed skills

This course assumes you already know cloud computing fundamentals — it drills Microsoft's terminology and exam patterns, not cloud fundamentals from zero.

Domain 1 · Describe cloud concepts 25–30%

Describe cloud computing

  • Define cloud computing; describe the shared responsibility model.
  • Define cloud models — public, private, hybrid — and identify appropriate use cases for each.
  • Describe the consumption-based model; compare cloud pricing models (CapEx vs. OpEx).
  • Describe serverless.

Describe the benefits of using cloud services

  • High availability and scalability.
  • Reliability and predictability.
  • Security and governance.
  • Manageability.

Describe cloud service types

  • IaaS, PaaS, SaaS — and appropriate use cases for each.

Domain 2 · Describe Azure architecture and services 35–40%

Describe the core architectural components of Azure

  • Regions, region pairs, and sovereign regions; availability zones; datacenters.
  • Resources and resource groups; subscriptions; management groups.
  • The hierarchy of resource groups, subscriptions, and management groups.

Describe Azure compute and networking services

  • Compare compute types: containers, virtual machines, functions.
  • VM options: Azure VMs, Virtual Machine Scale Sets, availability sets, Azure Virtual Desktop.
  • Resources required for VMs; app hosting options (web apps, containers, VMs).
  • Virtual networking: VNets, subnets, peering, Azure DNS, VPN Gateway, ExpressRoute.
  • Public and private endpoints.

Describe Azure storage services

  • Compare Azure Storage services; storage tiers; redundancy options.
  • Storage account options and storage types.
  • Moving files: AzCopy, Azure Storage Explorer, Azure File Sync.
  • Migration options: Azure Migrate, Azure Data Box.

Describe Azure identity, access, and security

  • Microsoft Entra ID and Microsoft Entra Domain Services.
  • Authentication methods: SSO, MFA, passwordless; external identities.
  • Microsoft Entra Conditional Access; Azure role-based access control (RBAC).
  • Zero Trust; defense-in-depth; Microsoft Defender for Cloud.

Domain 3 · Describe Azure management and governance 30–35%

Describe cost management in Azure

  • Factors affecting cost; the pricing calculator.
  • Cost management capabilities; the purpose of tags.

Describe features and tools for governance and compliance

  • Microsoft Purview; Azure Policy; resource locks.

Describe features and tools for managing and deploying Azure resources

  • The Azure portal; Azure Cloud Shell, CLI, and PowerShell.
  • Azure Arc; infrastructure as code (IaC); ARM templates.

Describe monitoring tools in Azure

  • Azure Advisor; Azure Service Health.
  • Azure Monitor, including Log Analytics, alerts, and Application Insights.
Where the points are Domain 2 is the largest single share, and its center of gravity is vocabulary mapping — compute options, networking primitives, storage tiers/redundancy, and the Entra ID/RBAC split. If you already know AWS or GCP, this is mostly relabeling what you know, not new material. Domain 3's tool-boundary questions (Policy vs. RBAC vs. locks; Advisor vs. Service Health vs. Monitor) are cheap, memorizable points — bank them all.
