DP-900 · Reference

Exam Blueprint

The authoritative skills-measured outline, straight from Microsoft. Everything testable lives in these four domains.

Primary source   Official Study Guide — Exam DP-900 · Skills as of 2026-07-21 · Page updated 2026-06-23.

Pass mark

700 / 1000. A scaled score — not a raw percentage. Aim for ~85%+ on practice to clear it comfortably.

Format

~40–60 questions, ~45 min. Multiple-choice, multi-select, drag-and-drop, and build-list. No live labs.

Assumed skills

Familiarity with relational vs. non-relational data, and transactional vs. analytical workloads. No coding required — mostly GA features.

Domain 1 · Describe core data concepts 25–30%

Vendor-neutral fundamentals. Know the kinds of data, where they live, and who works with them.

Describe ways to represent data

  • Features of structured data (fixed schema, rows & columns — relational tables).
  • Features of semi-structured data (self-describing tags/keys — JSON, XML, key-value).
  • Features of unstructured data (no schema — images, audio, video, free text, PDFs).

Identify options for data storage

  • Common file formats: delimited (CSV/TSV), JSON, XML, and columnar (Parquet, Avro, ORC).
  • Features of common data stores including databases (relational, document, key-value, graph, column-family).
  • Match an Azure datastore to a use case (SQL Database, Cosmos DB, Blob, Files, Table).

Describe common data workloads

  • Transactional (OLTP): many small reads/writes, normalized, ACID, current operational data.
  • Analytical (OLAP): large aggregate reads, denormalized/star schema, historical, read-optimized.

Identify roles and responsibilities for data workloads

  • Database administrator: availability, performance, backup/restore, security of the database.
  • Data engineer: builds and operates pipelines — ingest, transform, integrate, load.
  • Data analyst: models, explores, and visualizes data to surface insight (Power BI).

Domain 2 · Identify considerations for relational data on Azure 20–25%

Relational fundamentals plus the Azure SQL family and open-source managed databases.

Describe relational concepts

  • Features of relational data: tables, rows, columns, keys (primary/foreign), relationships.
  • Normalization and why it is used (reduce duplication and update anomalies).
  • Common SQL statements: DDL (CREATE/ALTER/DROP), DML (SELECT/INSERT/UPDATE/DELETE), DCL (GRANT/REVOKE).
  • Common database objects: tables, views, stored procedures, indexes.

Describe relational Azure data services

  • The Azure SQL family: Azure SQL Database (PaaS), Azure SQL Managed Instance, SQL Server on Azure VMs (IaaS).
  • Azure managed services for open-source databases: Azure Database for MySQL, PostgreSQL, and MariaDB.

Domain 3 · Describe considerations for working with non-relational data on Azure 15–20%

The smallest domain: Azure Storage services and Cosmos DB.

Describe the capabilities of Azure storage

  • Azure Blob storage: object storage for unstructured data; access tiers (hot / cool / cold / archive).
  • Azure Files: managed SMB/NFS file shares in the cloud.
  • Azure Table storage: schemaless key-value/NoSQL store for large volumes of structured non-relational data.

Describe the capabilities and features of Azure Cosmos DB

  • Use cases: globally distributed, low-latency, elastic apps (IoT, retail, gaming) needing multi-region writes.
  • APIs: NoSQL (Core), MongoDB, Cassandra, Table, and Apache Gremlin (graph) — pick the API by data model / migration source.

Domain 4 · Describe an analytics workload on Azure 25–30%

Tied for the largest domain. Large-scale and real-time analytics, plus Power BI visualization.

Describe common elements of large-scale analytics

  • Considerations for data ingestion and processing — ETL vs. ELT, pipelines, orchestration.
  • Options for analytical data stores: data warehouses vs. data lakes vs. the lakehouse.
  • Microsoft cloud services for large-scale analytics: Azure Databricks and Microsoft Fabric.

Describe considerations for real-time data analytics

  • The difference between batch and streaming data.
  • Microsoft cloud services for real-time analytics (Azure Stream Analytics, Fabric Real-Time Intelligence).

Describe data visualization in Microsoft Power BI

  • Capabilities of Power BI (Desktop authoring, the service, reports and dashboards).
  • Features of data models in Power BI (tables, relationships, measures).
  • Choose an appropriate visualization for the data (bar/column, line, pie, scatter, map, KPI, table/matrix).
Where the points are Domains 1 and 4 are the heavyweights (25–30% each) — core concepts and analytics. Don't neglect the small Domain 3 (15–20%): it is only Azure Storage + Cosmos DB, so it is the cheapest domain to master fully. Relational (Domain 2) rewards knowing the Azure SQL family distinctions cold.
