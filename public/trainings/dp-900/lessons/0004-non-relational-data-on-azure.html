Non-Relational Data on Azure

The smallest, narrowest domain — just Azure Storage and Cosmos DB. Master it fully; it's the cheapest points on the exam.

Azure Storage — three services, three jobs

ServiceWhat it isUse when
Blob storageObject storage for unstructured dataImages, video, backups, data-lake files at scale
Azure FilesManaged SMB/NFS file sharesYou need a mountable network drive
Table storageSchemaless key-value NoSQL storeLarge volumes of structured, non-relational data, cheaply
The one-word discriminator Objects → Blob. Mountable share → Files. Key-value → Table.

Blob access tiers

Blob storage prices access against latency. Colder = cheaper to store, slower/costlier to read.

TierUse for
HotFrequently accessed data
CoolInfrequent access, stored ≥ 30 days
ColdRarely accessed, stored ≥ 90 days
ArchiveAlmost never accessed; must be rehydrated before reading — cheapest storage

Azure Cosmos DB

Cosmos DB is a globally distributed, multi-model NoSQL database with single-digit-millisecond latency and multi-region writes. Reach for it when an app must be global, elastic, and low-latency (IoT, retail, gaming).

You pick an API to match your data model or migration source:

APIData modelChoose when
NoSQL (Core)JSON documents (native)New apps; flexible document schemas
MongoDBDocumentsMigrating a MongoDB app
CassandraColumn-familyMigrating a Cassandra workload
TableKey-valueUpgrading from Azure Table storage
GremlinGraph (vertices/edges)Highly connected/graph data
Match the API to the source Most Cosmos DB questions are "migrating from X → use the X API." NoSQL (Core) is the native default for new JSON apps.
