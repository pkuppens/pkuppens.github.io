The shape of the exam

DP-900 has four domains. The weighting tells you where to spend cram time:

DomainWeightWhat it is
D1 Describe core data concepts25–30%Vendor-neutral fundamentals: data types, workloads, file formats, data roles.
D2 Relational data on Azure20–25%Relational concepts, SQL, and the Azure SQL family + open-source databases.
D3 Non-relational data on Azure15–20%Azure Storage (Blob, Files, Table) and Cosmos DB.
D4 Analytics workload on Azure25–30%Large-scale and real-time analytics, and Power BI visualization.

Pass mark is 700/1000. Full details in the Exam Blueprint; vocabulary in the Glossary.

Where the effort goes Domains 1 and 4 carry the most weight (25–30% each). Domain 3 is the smallest (15–20%) and narrowest — just Azure Storage + Cosmos DB — so it's the cheapest domain to fully master. Learn the trap pairs (OLTP/OLAP, ETL/ELT, warehouse/lake) as contrasts.

Diagnostic (15 questions)

Answer honestly — guessing wastes the signal. Each answer reveals the explanation immediately. When you finish, note which tags (D1–D4) you missed, and drill those lessons first.
